# 0004 Build Attribution And Alerts

来源：`docs/0011-monitoring-review/design.md`、`docs/0011-monitoring-review/progress.md`

## 目标

实现收益归因和告警处理，让复盘可以解释收益来源和异常原因。

## 前置依赖

- 每日报告已完成。
- 持仓、交易、行业和成本数据可用。

## 执行步骤

1. 实现个股收益贡献。
2. 实现亏损贡献分析。
3. 实现行业贡献分析。
4. 实现风格因子贡献分析。
5. 实现交易成本影响分析。
6. 实现页面内告警列表。
7. 实现告警确认。
8. 实现告警关闭。
9. 将微信、钉钉、邮件告警作为后续延后项。

## 产出

- 收益归因能力。
- 页面内告警。
- 告警确认和关闭流程。

## 验收

- 可以看到收益来自哪些股票和行业。
- 告警可以被确认和关闭。
- 第一版不依赖外部消息渠道也能完成告警闭环。
