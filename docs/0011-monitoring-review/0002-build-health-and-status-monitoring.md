# 0002 Build Health And Status Monitoring

来源：`docs/0011-monitoring-review/design.md`、`docs/0011-monitoring-review/progress.md`

## 目标

实现数据源、策略、回测、模拟交易和实盘交易的状态监控。

## 前置依赖

- 各模块能输出任务状态、事件或日志。

## 执行步骤

1. 实现数据源状态监控，记录最近更新时间、失败次数和延迟。
2. 实现策略运行状态监控，记录是否按时生成信号、耗时和错误日志。
3. 实现回测任务状态监控，记录任务状态、耗时和失败原因。
4. 实现模拟交易状态监控，记录净值、持仓、订单和成交状态。
5. 实现实盘交易状态监控，记录订单失败、撤单、成交回报和紧急停止。
6. 实现风控拦截统计，记录拦截次数和原因。
7. 数据源超时生成 warning 级别告警。
8. 实盘订单失败生成 critical 级别告警。

## 产出

- 系统健康检查。
- 各模块运行状态监控。
- 告警生成规则。

## 验收

- 可以查看系统健康状态。
- 数据源、策略、回测和交易异常能形成事件。
- 告警级别与事件严重度匹配。
