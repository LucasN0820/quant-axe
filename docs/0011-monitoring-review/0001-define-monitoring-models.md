# 0001 Define Monitoring Models

来源：`docs/proposal.md`、`docs/total_progress.md`、`docs/0011-monitoring-review/design.md`

## 目标

定义健康检查、系统事件、告警事件和每日报告模型，为监控复盘打基础。

## 前置依赖

- 数据、策略、回测、模拟交易模块逐步提供运行记录。

## 执行步骤

1. 定义健康检查接口返回结构。
2. 定义系统事件模型，记录来源、级别、消息和时间。
3. 定义告警事件模型，支持 `open`、`acknowledged`、`resolved` 状态。
4. 定义每日报告模型，包含收益、基准、回撤、贡献、风险事件和生成时间。
5. 设计事件来源枚举：数据源、策略、回测、模拟交易、实盘交易、风控。
6. 设计告警确认和关闭流程。

## 产出

- 健康检查结构。
- 系统事件模型。
- 告警事件模型。
- 每日报告模型。

## 验收

- 异常事件可以形成告警并被确认。
- 报告结构能表达收益、回撤、持仓、交易和异常。
- 监控事件可按来源查询。
