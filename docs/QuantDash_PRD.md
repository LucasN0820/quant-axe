# 产品需求文档 (PRD)：个人量化看板 (QuantDash v1.0)

## 1. 项目背景与愿景

**项目目标**：为个人投资者（全栈开发背景）构建一个轻量级、响应式的量化工作台。
**核心价值**：通过技术手段整合分散的行情、新闻及舆情数据，辅助进行基于数据的理性决策，而非盲目跟风。

---

## 2. 核心功能模块 (Features)

### 2.1 股票自选池 (Watchlist)

- **搜索添加**：支持 A 股代码（6 位数字）及拼音首字母搜索。
- **数据持久化**：使用 `localStorage`（MVP阶段）或 `PostgreSQL` 存储用户自选列表。
- **实时概览**：列表页展示当前价格、涨跌幅、成交额等核心快照数据。

### 2.2 深度行情分析 (Market Insight)

- **交互式 K 线**：集成 ECharts，支持日线/周线切换，包含 MA（均线）、量比等指标。
- **动态分时图**：实时渲染当日价格波动的 Ticks 数据。
- **盘口数据**：展示买卖五档挂单及实时逐笔成交记录。

### 2.3 舆情与数据采集层 (Intelligence)

- **热搜聚合**：定时抓取微博、抖音、雪球等平台的财经热点关键词。
- **新闻瀑布流**：个股关联新闻的实时抓取与展示。
- **财务卡片**：聚合 PE (TTM)、PB、ROE、毛利率等关键财报指标。

---

## 3. 技术架构 (Tech Stack)

### 3.1 前端 (Frontend)

- **框架**: Next.js 15 (App Router)
- **样式**: Tailwind CSS + Shadcn UI
- **可视化**: Apache ECharts

### 3.2 后端 (Backend/Data)

- **BFF 层**: Next.js Route Handlers (处理前端请求与鉴权)
- **数据中台**: Python (FastAPI) + AkShare (负责爬取与清洗金融数据)
- **数据库**: PostgreSQL (存储用户信息及静态配置)
- **缓存**: Redis (存放高频刷新的实时行情)

## 5. 数据字典与接口定义 (Data Schema)

### 5.1 实时行情接口 (Real-time)

- **Endpoint**: `/api/stock/quote/[symbol]`
- **Frequency**: 3s - 5s 轮询
- **Fields**: `current_price`, `change_rate`, `volume`, `high`, `low`

### 5.2 历史序列接口 (History)

- **Endpoint**: `/api/stock/kline/[symbol]?type=daily`
- **Fields**: `date`, `open`, `close`, `high`, `low`, `volume`
