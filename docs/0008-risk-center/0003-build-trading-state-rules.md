# 0003 Build Trading State Rules

来源：`docs/proposal.md`、`docs/total_progress.md`、`docs/0008-risk-center/design.md`

## 目标

实现交易状态和数据质量相关风控规则，拦截停牌、涨跌停、流动性不足和异常数据。

## 前置依赖

- `data-center` 提供停牌、涨跌停、成交额和数据质量状态。
- 风控模型和事件记录已完成。

## 执行步骤

1. 实现流动性限制，订单金额超过成交额比例则阻塞。
2. 实现停牌拦截。
3. 实现涨停买入拦截。
4. 实现跌停卖出拦截。
5. 实现关键价格缺失时的数据异常拦截。
6. 实现数据源异常时的全局阻塞或警告。
7. 将所有拦截写入风控事件。

## 产出

- 流动性规则。
- 停牌和涨跌停规则。
- 数据异常拦截规则。

## 验收

- 风控可以拦截停牌、涨停买入、跌停卖出。
- 数据异常时系统能阻止策略继续下单。
- 拦截结果可以被监控复盘读取。
