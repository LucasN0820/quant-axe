# 0001 Define Backtest Data Models

来源：`docs/proposal.md`、`docs/total_progress.md`、`docs/0006-backtest-center/design.md`

## 目标

定义回测配置、任务、交易明细、每日持仓和净值曲线模型，为日频回测引擎打基础。

## 前置依赖

- `data-center` 提供交易数据。
- `universe-center` 提供股票池快照。
- `research-lab` 提供策略版本和信号。

## 执行步骤

1. 定义 `BacktestConfig`，包含策略、股票池、日期、资金、基准、手续费、滑点和仓位限制。
2. 定义回测任务模型，记录状态、进度、开始结束时间、错误和数据版本。
3. 定义交易明细模型，记录成交股票、方向、数量、价格、费用和原因。
4. 定义每日持仓模型，记录日期、股票、数量、市值和权重。
5. 定义净值曲线模型，记录现金、持仓市值、总资产、收益和回撤。
6. 锁定数据版本、股票池版本和策略版本，保证回测可复现。

## 产出

- 回测配置模型。
- 回测任务模型。
- 交易、持仓、净值模型。
- 版本锁定规则。

## 验收

- 同一配置能锁定同一组输入版本。
- 回测结果可以关联到策略、股票池和数据版本。
- 结果结构能支撑报告接口。
