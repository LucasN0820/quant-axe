# 0002 Build Daily Backtest Engine

来源：`docs/0006-backtest-center/design.md`、`docs/0006-backtest-center/progress.md`

## 目标

实现 A 股日频事件驱动回测主循环，逐日加载可见数据、生成信号、构建组合并模拟成交。

## 前置依赖

- 回测数据模型已完成。
- 策略预览、组合目标持仓和基础行情数据可用。

## 执行步骤

1. 创建回测任务后锁定数据、股票池和策略版本。
2. 从 `start_date` 到 `end_date` 按交易日逐日推进。
3. 每个交易日只加载当日及以前可见数据。
4. 调用策略生成信号。
5. 调用组合模块生成目标仓位。
6. 调用风控或执行模拟器检查交易约束。
7. 根据目标仓位生成买卖指令。
8. 执行模拟成交，更新现金、持仓和净值。
9. 记录每日日志、持仓和交易明细。

## 产出

- 日频回测主循环。
- 账户现金和持仓账本。
- 每日状态记录。

## 验收

- 可以使用小样本跑完整个回测循环。
- 回测过程中不能读取未来日期数据。
- 重复运行同一配置结果一致。
