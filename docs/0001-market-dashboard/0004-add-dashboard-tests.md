# 0004 Add Dashboard Tests

来源：`docs/0001-market-dashboard/design.md`、`docs/0001-market-dashboard/progress.md`

## 目标

为行情看板补齐组件测试、store 测试和浏览器端 E2E 测试，保障后续数据接入不破坏现有体验。

## 前置依赖

- 行情看板组件已拆分。
- quote 轮询和降级逻辑已集中管理。

## 执行步骤

1. 使用 mock BFF 接口测试 UI 组件，不依赖真实行情源。
2. 测试自选股添加、删除、去重、选中和持久化恢复。
3. 使用固定 `KlinePoint[]` 测试 K 线图容器可正常挂载。
4. 测试 quote 失败时展示降级状态。
5. 测试新闻、公告、财务、舆情为空时展示空状态。
6. 增加浏览器端 E2E：添加股票、切换股票、切换 K 线周期、刷新页面后恢复自选股。

## 产出

- 前端组件测试。
- Zustand store 单元测试。
- 行情看板 E2E 测试。

## 验收

- 测试能在无真实外部网络的情况下稳定运行。
- 核心流程失败时能给出明确断言。
- 新增测试覆盖当前 `progress.md` 中待完成的测试项。
