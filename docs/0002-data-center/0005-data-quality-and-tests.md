# 0005 Data Quality And Tests

来源：`docs/proposal.md`、`docs/total_progress.md`、`docs/0002-data-center/design.md`

## 目标

建立数据质量检查和独立测试，确保 clean、adjusted、serving 层可靠。

## 前置依赖

- 核心数据表和服务接口已建立。
- 数据更新任务可运行。

## 执行步骤

1. 实现缺失值检查，重点覆盖价格、成交量、成交额、交易日历。
2. 实现重复交易日检查。
3. 实现异常涨跌幅检查。
4. 实现成交量和成交额异常检查。
5. 实现 source 和 updated_at 完整性检查。
6. Provider adapter 使用固定原始响应 fixture 测试解析。
7. 清洗任务使用小样本测试去重、字段类型和缺失值处理。
8. API 层使用测试数据库，不依赖真实外部网络。
9. 数据质量规则可单独运行并输出报告。

## 产出

- 数据质量规则集。
- 后端单元测试。
- 数据质量报告输出。

## 验收

- 数据源异常不会污染 clean 层。
- 质量规则能定位异常原因。
- 所有核心服务接口测试不依赖外部网络。
