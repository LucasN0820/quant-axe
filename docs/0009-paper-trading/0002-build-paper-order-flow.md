# 0002 Build Paper Order Flow

来源：`docs/0009-paper-trading/design.md`、`docs/0009-paper-trading/progress.md`

## 目标

实现模拟订单创建、撤单、风控检查和状态流转。

## 前置依赖

- 模拟交易模型已完成。
- 风控接口可用。

## 执行步骤

1. 实现账户创建接口。
2. 实现模拟订单创建。
3. 创建订单前调用风控检查。
4. 风控拒绝时订单标记为 `rejected`，不进入成交。
5. 实现模拟撤单，只有未成交或部分成交订单可撤。
6. 记录订单状态：`pending`、`filled`、`partial_filled`、`cancelled`、`rejected`。
7. 记录策略日志和订单原因，方便复盘。

## 产出

- 虚拟账户创建能力。
- 模拟订单创建和撤单能力。
- 风控接入。

## 验收

- 风控拒绝的订单不会成交。
- 撤单后订单状态正确。
- 策略每日可生成模拟调仓订单。
