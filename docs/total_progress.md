# QuantDash 全系统实现进度

文档来源：`docs/proposal.md`

更新时间：2026-05-19

## 1. 总体原则

QuantDash 的实现顺序遵循“先数据、再研究、再验证、再执行”的路径：

```text
行情看板
  -> 数据中心
  -> 股票池
  -> 因子引擎
  -> 策略研究
  -> 回测中心
  -> 组合中心
  -> 风控中心
  -> 模拟交易
  -> 半自动实盘
  -> 监控复盘
```

核心约束：

- 不在数据质量稳定前推进复杂策略。
- 不在回测可信前推进模拟交易。
- 不在模拟交易稳定前推进实盘。
- 第一版实盘坚持人工确认，不做全自动下单。
